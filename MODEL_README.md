# ML-детектор двигателя мопеда

## 🎯 Что это такое?

Система распознавания звука двигателя мопеда на основе **классического машинного обучения** (RandomForest).

Алгоритм извлечения признаков вдохновлён проектом [SkufUp](https://github.com/rudnstudent/SkufUp) — детектором звука открытия пивной банки.

---

## 📖 ML vs ИИ — в чём разница?

Это частый вопрос! Давайте разберёмся:

### 🤖 Искусственный интеллект (ИИ / AI)

**ИИ** — это **общий термин** для любых систем, которые имитируют человеческий интеллект.

```
ИИ (Artificial Intelligence)
  ├── Машинное обучение (ML)
  ├── Глубокое обучение (Deep Learning)
  ├── Экспертные системы
  ├── Эволюционные алгоритмы
  └── И т.д.
```

**Примеры ИИ:**
- Шахматный компьютер (экспертная система)
- Навигатор (поиск пути)
- Голосовой помощник (распознавание речи)

---

### 📊 Машинное обучение (ML / Machine Learning)

**ML** — это **подраздел ИИ**, где компьютер **учится на данных**, а не выполняет жёсткие правила.

```
ML (Machine Learning)
  ├── Классические алгоритмы
  │   ├── RandomForest (случайный лес) ← ИСПОЛЬЗУЕМ МЫ
  │   ├── SVM (метод опорных векторов)
  │   ├── KNN (ближайшие соседи)
  │   └── Linear Regression
  └── Ансамбли
      ├── Gradient Boosting
      └── XGBoost
```

**Как работает ML:**
```
Данные (аудио) → Извлечение признаков → Обучение модели → Предсказание
```

**Пример:**
- Обучаем на 1000 записей двигателя и 2000 записей без него
- Модель находит паттерны в частотах
- Предсказывает: "это двигатель" или "нет"

---

### 🧠 Глубокое обучение (Deep Learning / Нейросети)

**DL** — это **подраздел ML**, использующий нейронные сети с множеством слоёв.

```
Deep Learning
  ├── CNN (свёрточные сети) — YAMNet
  ├── RNN/LSTM (рекуррентные сети)
  ├── Transformers
  └── Autoencoders
```

**Примеры:**
- YAMNet (распознавание 521 класса звуков)
- ChatGPT (генерация текста)
- Распознавание лиц

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install scikit-learn numpy scipy soundfile
```

### 2. Подготовка данных

Создайте две папки:

```
moped_samples/     ← Записи с двигателем (10+ файлов)
noise_samples/     ← Записи без двигателя (20+ файлов)
```

**Где взять данные:**

- **Записать с камеры** (30 сек каждый):
  ```bash
  python rtsp_audio_recorder.py rtsp://камера/stream -d 30 -n 10
  ```

- **Записать фоновые шумы:**
  - Тишина на улице
  - Разговоры людей
  - Шум ветра
  - Птицы
  - Машины (не мопед)

- **Скачать датасет ESC-50:**
  ```
  https://github.com/karolpiczak/ESC-50
  ```
  (класс "Engine" = 100 записей двигателя)

### 3. Обучение модели

```bash
python train_moped_detector.py --moped moped_samples/ --noise noise_samples/
```

**Результат обучения:**
```
📊 ИТОГО:
   Примеров: 1500
   Двигатель: 500 (33.3%)
   Без двигателя: 1000 (66.7%)

📊 РЕЗУЛЬТАТЫ НА TEST SET:
   Точность: 0.9456 (94.6%)

💾 Модель сохранена: moped_model.pkl
```

### 4. Использование модели

```bash
# Анализ папки с записями
python moped_ml_detector.py -i rtsp_recordings

# Анализ одного файла
python moped_ml_detector.py -f audio_20260410_123456.wav
```

## 📊 Извлекаемые признаки

Модель использует **15 числовых признаков**:

| Признак | Описание | Важность |
|---------|----------|----------|
| `rms` | Громкость (среднеквадратичное) | ⭐⭐⭐ |
| `peak` | Пиковое значение | ⭐⭐ |
| `zero_crossing_rate` | Частота пересечения нуля | ⭐⭐⭐ |
| `energy_bass` | Энергия басов (60-250 Hz) | ⭐⭐⭐⭐⭐ |
| `energy_mid` | Энергия средних (500-2000 Hz) | ⭐⭐⭐⭐ |
| `energy_high` | Энергия высоких (4000-8000 Hz) | ⭐⭐⭐ |
| `spectral_centroid` | Центр тяжести спектра | ⭐⭐⭐⭐ |
| `hl_ratio` | Соотношение высоких/низких | ⭐⭐⭐⭐ |
| `spectral_flux` | Неровность спектра | ⭐⭐⭐ |
| `snr_estimate` | Отношение сигнал/шум | ⭐⭐ |

## 💻 Требования к железу

### Минимальные (работает на всём)

| Компонент | Требование |
|-----------|------------|
| **CPU** | Любной 2-ядерный (Intel i3 / AMD Ryzen 3) |
| **RAM** | 2 GB |
| **Диск** | 100 MB |
| **GPU** | ❌ Не нужен! |

**Реальные цифры:**
- **Обучение**: 5-30 секунд на 1000 примеров
- **Предсказание**: 10-50 мс на 1 секунду аудио
- **Память модели**: ~1-5 MB (файл .pkl)

### Рекомендуемые (для работы в реальном времени)

| Компонент | Требование |
|-----------|------------|
| **CPU** | 4-ядерный (Intel i5 / AMD Ryzen 5) |
| **RAM** | 4 GB |
| **Диск** | 500 MB |
| **GPU** | ❌ Не нужен! |

**Почему не нужна видеокарта:**
- RandomForest работает на CPU
- Не использует матричные вычисления
- Потребляет < 100 MB RAM при работе

## 🎯 Когда что использовать?

### Частотный детектор (`moped_frequency_detector.py`)

✅ **Используйте если:**
- Нужно быстро и просто
- Частоты двигателя известны и постоянны
- Нет данных для обучения
- Не нужна высокая точность

❌ **Не используйте если:**
- Частоты перекрываются с другими звуками
- Много фонового шума
- Нужна высокая точность

---

### ML-детектор (`moped_ml_detector.py`)

✅ **Используйте если:**
- Нужна высокая точность (90%+)
- Есть данные для обучения
- Звук двигателя меняется (обороты, расстояние)
- Нужно отличать от похожих звуков

❌ **Не используйте если:**
- Нет данных для обучения
- Нужно "здесь и сейчас"
- Нет времени на настройку

---

### YAMNet (`moped_detector.py`)

✅ **Используйте если:**
- Нужно распознавать много типов звуков
- Нет данных для обучения (модель предобучена)
- Есть ресурсы (CPU/RAM)

❌ **Не используйте если:**
- Нужна высокая скорость
- Ограниченные ресурсы
- Нужен только один тип звука

## ❓ FAQ

### Q: Сколько данных нужно для обучения?

**Минимум:**
- 10 файлов с двигателем (по 10-30 сек)
- 20 файлов без двигателя

**Идеально:**
- 50+ файлов с двигателем
- 100+ файлов без двигателя

### Q: Можно ли использовать предобученную модель?

Да! Модель YAMNet уже предобучена. Но для RandomForest нужно обучать на своих данных.

### Q: Почему не использовать нейросеть?

- Нейросети требуют GPU для быстрого обучения
- RandomForest работает на CPU
- Для 1 класса звука нейросеть — overkill
- RandomForest объясним (видно важность признаков)

### Q: Как улучшить точность?

1. **Больше данных** (главный фактор!)
2. **Разнообразные данные** (разные условия, расстояние, обороты)
3. **Больше деревьев** (`--trees 200`)
4. **Качественные негативные примеры** (разные типы шума)
